# AI测试用例自动生成器

基于DeepSeek AI的智能测试用例生成工具，支持需求识别、分解和测试用例自动生成。

## 功能特性

### 🧠 AI智能需求处理
- **智能需求识别**: 使用AI大模型自动识别文档中的软件需求，避免简单的文本分割
- **需求类型分析**: 自动识别功能需求、性能需求、安全需求等不同类型
- **智能需求分解**: 将复杂需求自动分解为可测试的子需求
- **优先级评估**: 自动评估需求的测试优先级

### 📊 批量处理能力
- 支持Excel、Word、PDF、TXT等多种文件格式
- 批量生成测试用例，支持并行处理
- 自动识别需求列和需求段落

### 🎯 测试用例生成
- 生成正向测试用例、异常测试用例、边界测试用例
- 支持自定义测试用例模板
- 导出Excel和CSV格式

## 安装要求

```bash
pip install streamlit pandas openpyxl python-docx openai PyPDF2
```

## 使用方法

1. **配置API Key**
   - 在侧边栏输入DeepSeek API Key
   - 或设置环境变量 `DEEPSEEK_API_KEY`

2. **单条需求处理**
   - 在"单条需求"标签页输入功能需求
   - 点击生成测试用例

3. **批量需求处理**
   - 在"批量（Excel/Word）"标签页上传文件
   - 启用AI需求识别和分解功能
   - 执行AI需求分析
   - 批量生成测试用例

## AI需求处理功能

### 智能需求识别
- 自动识别文档中的独立需求单元
- 忽略格式标记和非需求内容
- 支持复杂文档结构分析

### 需求分解
- 自动判断需求复杂度
- 将复杂需求分解为可测试子需求
- 保持需求逻辑完整性

### 类型分类
- 功能需求
- 性能需求  
- 安全需求
- 易用性需求
- 兼容性需求
- 可靠性需求

## 文件格式支持

- **Excel (.xlsx)**: 自动检测需求列
- **Word (.docx)**: AI智能识别需求段落
- **PDF (.pdf)**: AI智能提取和识别需求
- **文本文件 (.txt, .csv)**: AI智能分析需求内容

## 配置选项

### AI处理选项
- 启用/禁用AI需求分析
- 启用/禁用AI需求分解
- 调整AI模型温度参数

### 测试用例配置
- 自定义测试用例列名
- 设置各类测试用例数量
- 配置并行处理数量

## 使用示例

```python
# 启动应用
streamlit run app.py
```

## 注意事项

- 确保网络连接正常，能够访问DeepSeek API
- 大文件处理可能需要较长时间
- 建议先进行AI需求分析，确认识别结果后再生成测试用例

## 故障排除

1. **API连接失败**: 检查API Key和网络连接
2. **文件解析错误**: 确认文件格式正确且未损坏
3. **AI识别失败**: 尝试调整温度参数或简化需求描述
